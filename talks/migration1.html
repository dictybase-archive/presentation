Development environment

  • Three tier setup

    • Development, staging and production

    Unix environments, virtual machine based (at least staging and prod)

  • Software versioning

    • Git in github and private git repo

    Software release cycle follow gitflow workflow

    Cvs for legacy code(will retire)

  • Database release cycle

    • Approx at least three releases per year

Developmental practices(Not buzzwords)

  • Modular and loosely connected softwares

  • OO practice with reusable parts

  • Unit testing

  • MVC web applications

  • Command line applications

  • API documentation

NOT RECOMMENDED practices(Seriously)

  • CGI scripts with modperl

  • Lack of unit test

  • Ad-hoc scripts
